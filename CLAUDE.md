## Architecture

CODESYS REST API wrapper: HTTP/CLI → ActionService → IronPython script generation → named pipe transport → CODESYS process (IronPython 2.7 runtime).

### Request flow

```
HTTP request or CLI invocation
  → http_server.py / cli_entry.py
    → action_layer.py (ActionService.execute)
      → ironpython_script_engine.py (generates IronPython 2.7 script)
        → script_executor.py (sends via named pipe, waits for result)
          → named_pipe_transport.py (Windows kernel32 ctypes)
            → CODESYS process running assets/PERSISTENT_SESSION.py
```

### Two Python runtimes

- **Host side** (Python 3.13+): everything under `src/codesys_api/`
- **CODESYS side** (IronPython 2.7): `src/codesys_api/assets/PERSISTENT_SESSION.py` — runs inside CODESYS, reads scripts from named pipe, executes via `scriptengine` module, returns JSON results

### Key modules

| Module | Role |
|--------|------|
| `action_layer.py` | Routes `ActionType` enum to engine methods; owns `ActionRequest`/`ActionResult` |
| `ironpython_script_engine.py` | Generates IronPython 2.7 scripts from action params; returns `ExecutionSpec(script, timeout)` |
| `proven_primitives.py` | One function per probe-verified CODESYS scriptengine primitive — enforces boundary contract |
| `codesys_process.py` | `CodesysProcessManager`: launch/stop/monitor CODESYS subprocess |
| `script_executor.py` | Sends script over named pipe, reads JSON result with timeout |
| `named_pipe_transport.py` | Low-level Windows named pipe I/O via ctypes/kernel32 |
| `server_logic.py` | Request validation, response normalization, status payload builders |
| `app_runtime.py` | Wires together process manager, transport, executor, action service |

## Boundary Contract

**Critical rule**: Only CODESYS scriptengine primitives that have been validated by a real-CODESYS probe may be used in product code. See `docs/CODESYS_BOUNDARY_CONTRACT.md`.

`proven_primitives.py` is the code representation of this contract. Each function returns a single-line IronPython snippet backed by a named passing probe. `ironpython_script_engine.py` must compose scripts from this module.

## IronPython Script Generation

Scripts generated by `ironpython_script_engine.py` run inside CODESYS as IronPython 2.7. Key rules:

- Scripts embedded in Python `.format()` strings need `{{`/`}}` for literal braces (dict literals, set comprehensions)
- Scripts NOT using `.format()` must use literal `{`/`}` — double braces become syntax errors in IronPython
- Always use `_escape()` from `proven_primitives.py` for user-provided strings
- The `scriptengine` module is available globally inside CODESYS (no import needed)
- `system` object provides `get_messages()`, `delay()`, and other runtime utilities

## Testing

- **Unit tests** (`tests/unit/`): Mock-based with `FakeProcessManager`, `FakeScriptExecutor`, `FakeEngineAdapter`
- **Integration** (`tests/integration/`): Real named pipes
- **E2E mock** (`tests/e2e/mock/`): HTTP server without CODESYS (`@pytest.mark.mock`)
- **E2E real** (`tests/e2e/codesys/`): Requires real CODESYS (`@pytest.mark.codesys`, `CODESYS_E2E_ENABLE=1`)
- **Manual probes** (`scripts/manual/`): Boundary-layer validation against real CODESYS

## Environment Variables

Required for real CODESYS:
- `CODESYS_API_CODESYS_PATH` — path to CODESYS.exe
- `CODESYS_API_CODESYS_PROFILE` — profile name
- `CODESYS_API_CODESYS_PROFILE_PATH` — path to .profile.xml
- `CODESYS_API_CODESYS_NO_UI` — set to `1` for headless mode (required for compile to work reliably)

Local overrides in `.env.real-codesys.local` (gitignored).
